STATS CORRELATION Extension Command

Calculate Pearson correlations and confidence intervals. The correlations are actually calculated by the built-in CORRELATIONS procedure, the output from which is used in calculating the confidence intervals.

STATS CORRELATIONS VARIABLES=variable list*

/WITH
VARIABLES=variable list

/OPTIONS
CONFLEVEL = confidence level as percentage
METHOD = FISHER** or BOOTSTRAP

/MISSING
PAIRWISE=YES** or LISTWISE=YES
EXCLUDE=YES** or INCLUDE=YES

/HELP

* Required
** Default

STATS CORRELATIONS /HELP. prints this information and does nothing else.

STATS CORRELATIONS VARIABLES=x y z
/WITH VARIABLES=w
/OPTIONS CONFLEVEL=90.

The first VARIABLES keyword and VARIABLES on the WITH subcommand specify the variables to be correlated. If only the first VARIABLES keyword is used, a square correlation matrix is calculated. If /WITH VARIABLES is also given, then only the correlations of the first set of variables with the second set are calculated. For example, if there are five variables in the first list and two in the second, ten correlations are produced. Note that WITH is a subcommand here rather than appearing within a variable list.

OPTIONS

CONFLEVEL specifies the level for the confidence interval as a percentage. The default is 95.

METHOD specifies how the intervals are calculated. FISHER uses the approximate normality of the Z transform of the correlation. BOOTSTRAP calculates bootstrap intervals. This requires that the Bootstrap option for Statistics be installed. Otherwise, an error is generated.

MISSING

This subcommand specifies how user missing values are treated. System missing values are always excluded. Missing value deletion can be PAIRWISE, which is the default, or LISTWISE. Note that these keywords require YES or NO, which is different from their usage in other procedures.

EXCLUDE is the default and excludes user missing values. INCLUDE includes user missing values as if they were valid values.
